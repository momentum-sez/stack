# v0.4.36 Patchlist

## Highlights

- **Jurisdiction scope hints** for SmartAssetReceipt (`jurisdiction_scope`, `harbor_ids`, `harbor_quorum`).
- **Registry quorum policy** (`credentialSubject.quorum_policy`) with quorum-aware `mez asset compliance-eval`.
- **Portable rule evaluation evidence** as a first-class artifact type (`rule-eval-evidence`) with CLI scaffolding.

## Changes

### Schemas

- Extend `schemas/smart-asset.receipt.schema.json` with:
  - `jurisdiction_scope` (`all_active | subset | quorum`)
  - `harbor_ids` (list of relevant harbors)
  - `harbor_quorum` (optional threshold for `quorum` scope)

- Add rule evaluation evidence schemas:
  - `schemas/rule-eval-evidence.schema.json`
  - `schemas/rule-eval-evidence.attachment.schema.json`

- Extend `schemas/vc.smart-asset-registry.schema.json` with `credentialSubject.quorum_policy`.

### CLI / Tools

- `mez asset state receipt-init`:
  - `--jurisdiction-scope all_active|subset|quorum`
  - `--harbor-id <id>` (repeatable)
  - `--harbor-quorum <n>`

- `mez asset registry-init`:
  - `--quorum-policy <yaml|json>` to embed `credentialSubject.quorum_policy`.

- `mez asset compliance-eval`:
  - Evaluates **active bindings only** by default.
  - Applies `credentialSubject.quorum_policy` when present.
  - `--json` for machine-readable reporting (includes quorum decision).

- New: `mez asset rule-eval-evidence-init`
  - Generates a `MEZRuleEvaluationEvidence` object.
  - Optional Ed25519 signing (`--sign --key ...`) and CAS storage (`--store`).

### Artifact CAS / Strict verification

- Add strict semantic digest support for artifact type `rule-eval-evidence`:
  - `digest = sha256(JCS(evidence_without_proof))`

### Docs

- Expand `docs/roadmap/PRODUCTION_GRADE_SPEC.md` with:
  - Implemented v0.4.36 deliverables
  - Forward roadmap (trade corridors, capital corridors, arbitration)
