# v0.4.28 — Witness bundle attestation VC

This patch adds an optional provenance layer for artifact witness bundles.

## What changed

### New schema
- `schemas/vc.artifact-witness-bundle.schema.json` — validates `MSEZArtifactWitnessBundleCredential` VCs.

### New CLI commands
- `msez artifact bundle attest <bundle.zip> ...`
  - Reads `manifest.json` from a witness bundle.
  - Computes `manifest_digest_sha256 = SHA256(JCS(manifest.json))`.
  - Emits an optional signed VC binding an issuer DID to that manifest digest.
- `msez artifact bundle verify <bundle.zip> --vc <attestation.vc.json>`
  - Verifies that the bundle's `manifest.json` digest matches the VC subject.
  - Optionally verifies the VC signature(s) (default behavior).

### Documentation
- `spec/97-artifacts.md` extended with a Witness bundles section and attestation examples.
- `README.md` updated with new CLI examples.

### Tests
- `tests/test_artifact_bundle_attestation.py`
  - signed attestation roundtrip
  - unsigned attestation acceptance with `--no-require-proof`
  - tamper detection on manifest modifications

## Version
- Stack spec version bumped to **0.4.28**.
