This patch adds a **watcher attestation aggregator** so fork-like divergences can be detected quickly without transporting receipts.

## Added

- `msez corridor state watcher-compare`:
  - Ingest watcher attestation VCs (signed).
  - Compare `(receipt_count, final_state_root)` per corridor.
  - Strong fork signal: same `receipt_count` with different `final_state_root`.
  - Optional strictness:
    - `--fail-on-lag` to treat receipt-count divergence as failure (default: warn).
    - `--enforce-authority-registry` to require watcher signers are authorized by the corridor authority-registry chain (if present).
    - `--require-artifacts` to require committed checkpoint artifacts are resolvable via CAS (`dist/artifacts/checkpoint/<digest>.*`).

## Fixed

- `cmd_corridor_state_verify`: define `enforce_checkpoint_policy` before first use (previously could raise `UnboundLocalError`).
- `schema_validator(...)` call sites now pass `Path` values (previous `str(...)` usage could break execution).

## Version

- Stack spec version bumped to `0.4.16`.
